The `corde` command line runner has several useful options.
Some of the options shown below can also be used together to run tests exactly the way you want.
Some one of Corde's [Configuration](/docs/Configuration) options can also be specified through the CLI.

We are working to make all the configuration options available in CLI.

## Running from the command line

Run all tests (default):

```bash
corde
```

Run only the tests that were specified with a pattern or filename:

```bash
corde path/to/test.js
```

:::info
You can run `corde --help` to view all available options.
:::

## Available Commands

### `--config`

**Alias: `-c`**

Set config file path for the next execution. I.E:

<Tabs
  defaultValue="long"
  values={[
    { label: "Long", value: "long" },
    { label: "Alias", value: "alias" },
   ]
  }>
  <TabItem value="long">

```bash
corde --config ../corde.config.js
```

  </TabItem>
  <TabItem value="alias">

```bash
corde -c ../corde.config.js
```

  </TabItem>
</Tabs>

### `--files`

**Alias: `-f`**

Set the path for all tests. Use this if you wan to specify a single path. for Array, use only `corde path1 path2`

<Tabs
  defaultValue="single"
  values={[
    { label: "Single Files", value: "single" },
    { label: "Multiple Files", value: "multiple" },
   ]
  }>
  <TabItem value="single">

```bash
corde --files ./tests/test1.js
```

  </TabItem>
  <TabItem value="multiple">

```bash
corde --files ./tests/test1.js ./tests/test2.js
```

  </TabItem>
</Tabs>

Using alias:

```bash
corde -f ./tests/test1.js ./tests/test2.js
```

:::note
This is also a "anonymous" command. It means that you can just pass the files path without use `-f` or  
`--files`. I.E:

```bash
corde ./tests/test1.js
```

:::

---

### `--init`

**Alias: `-i`**  
**Default: `json`**  
**Options: `js json ts`**

Initialize a config (`corde.config.*`) file. The available file formats are: `.js`, `.json`, `.ts`.

<Tabs
  defaultValue="long"
  values={[
    { label: "Long", value: "long" },
    { label: "Alias", value: "alias" },
   ]
  }>
  <TabItem value="long">

```bash
corde --init js
```

  </TabItem>
  <TabItem value="alias">

```bash
corde -i js
```

  </TabItem>
</Tabs>

### `--validate`

**Alias: `-v`**

Search for corde configs and check if all required data is set. Also check if file paths are valid.
_This does not check if tokens are valid._

   ]
  }>
  <TabItem value="long">

```bash
corde --validate
```

  </TabItem>
  <TabItem value="alias">

```bash
corde -v
```

  </TabItem>
</Tabs>
